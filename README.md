# ABC Store Sales Analysis Dashboard

## About the Project

This project involved analysing ABC Store sales data to understand business performance and uncover insights related to sales, customers, products, employees, and pricing.

The goal was to take data stored in a relational database, use SQL to explore and analyse the information, and build an interactive Power BI dashboard that helps users understand key business trends and make data-driven decisions.

This project demonstrates my ability to work through the full data analytics process, from working with raw data and databases to creating meaningful visual reports.

## Tools & Technologies

* **Oracle SQL** – Data extraction, querying, and analysis
* **Power BI Desktop** – Dashboard creation and data visualisation
* **Power Query** – Data cleaning and transformation
* **DAX** – Creating measures and KPIs

## Dataset

The dataset was provided as an SQL script containing multiple related tables, including:

* Customers
* Orders
* Products
* Employees
* Departments
* Product pricing history

The data was loaded into an Oracle database, where SQL queries were used to analyse the data before connecting it to Power BI for reporting.

## Data Preparation

Before creating the dashboard, I performed data preparation tasks to ensure the data was suitable for analysis:

* Reviewed the database tables and relationships
* Checked and corrected data types
* Cleaned and transformed data using Power Query
* Created relationships between tables
* Prepared the data model for Power BI reporting

## Dashboard Overview

The dashboard provides an overview of ABC Store's business performance through several analysis areas:

### Key Performance Indicators

The dashboard highlights important metrics such as:

* Total Revenue
* Total Orders
* Total Customers
* Total Items Sold

### Product Analysis

This section focuses on product performance by identifying:

* Top-performing products
* Products contributing the most to sales
* Product sales comparisons

### Customer Analysis

Customer behaviour was analysed by looking at:

* Number of orders per customer
* Customer spending patterns
* High-value customers

### Employee & Department Analysis

Employee data was analysed to understand:

* Employee distribution across departments
* Workforce composition

### Pricing Analysis

The pricing section analyses product price history by comparing:

* Standard Price
* Minimum Price
* Price changes over time
